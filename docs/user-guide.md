# RecordWise 使用文档

RecordWise 是一个会议录音纪要 App。它可以录音或上传音频，按说话人生成原文，整理会议总结和待办事项，并对指定发言做翻译朗读。

## 1. 首次使用

1. 安装 APK。
2. 打开 App，进入底部导航的 `设置`。
3. 在 `后端服务地址` 填写自己的后端 URL。
4. 点击 `测试连接`。
5. 看到连接成功后点击 `保存`。

如果测试失败，先检查：

- 手机和后端是否在同一网络，或公网域名是否可访问。
- 后端是否已启动。
- 地址是否包含正确端口，例如 `:8000`。
- 后端 `/api/health` 是否显示 DashScope 和 OSS 已配置。

## 2. 首页

首页用于查看最近会议记录和进入主要功能。

- 最近会议：显示已保存的会议记录。
- 录音入口：进入现场录音或上传音频。
- 会议卡片：点击后进入会议详情页。

## 3. 录音和上传

进入底部导航 `录音`。

可选择：

- 主要语种：普通话、粤语、英语、日语、法语。
- 会议类型：会议、访谈、讲座、电话、聊天、头脑风暴等。
- 现场录音：点击麦克风开始，再次点击结束并处理。
- 上传音频：选择已有音频文件上传后处理。

处理流程：

1. App 将音频上传到后端。
2. 后端标准化音频并上传 OSS。
3. DashScope Fun-ASR / Paraformer 转写并做说话人分离。
4. Qwen 生成会议纪要和待办事项。
5. App 保存结果并进入详情页。

## 4. 会议详情

详情页顶部显示：

- 会议名称。
- 会议时间。
- 会议时长。
- 主要语种。
- 参与人数。

右上角功能：

- 编辑会议名称。
- 复制整份会议内容。

详情页有三个主页签：

- `原录音文本`
- `总结文本`
- `待办事项`

## 5. 原录音文本

原录音文本按说话人和时间显示。

支持：

- 查看每段发言时间。
- 查看自动识别的说话人。
- 修改说话人名称。
- 自动匹配说话人头像。
- 对指定发言进行翻译朗读。

翻译朗读当前支持：

- 英语
- 日语
- 粤语
- 普通话
- 法语

翻译朗读失败时，优先检查：

- 后端地址是否正确。
- DashScope Key 是否有效。
- Qwen-MT 和 Qwen-TTS 模型名称是否可用。
- 后端日志是否出现模型额度、权限或网络错误。

## 6. 总结文本

总结文本页签下还有两个文档视图：

- `总结文档`：AI 生成的会议纪要。
- `原文汇总`：按 `发言人、发言时间、发言内容` 汇总的完整原文文档。

原文汇总规则：

- 如果连续多段都是同一个发言人，会合并成同一段。
- 如果发言人变化，才新开一段。
- 这个文档只用于阅读和复制，不提供翻译功能。

总结文档支持：

- 编辑总结文本。
- 保存编辑。
- 复制总结文本。
- 导入总结模板。
- 切换默认模板或导入模板后重新生成总结。

模板导入流程：

1. 点击导入模板。
2. 粘贴模板文字。
3. App 分析模板结构。
4. 用户确认后保存模板。
5. 切换到导入模板并重新生成总结。

## 7. 待办事项

待办事项页展示会议中提取出的行动项。

支持：

- 查看待办内容。
- 查看负责人、截止时间和来源说话人。
- 勾选完成状态。
- 勾选后文字会显示删除线。
- 重点待办会有高亮标记。

重点待办判断规则：

同一事项被 3 个或以上不同说话人在会议中提到，才标记为重点待办。

## 8. 历史记录

进入底部导航 `记录`。

支持：

- 查看历史会议。
- 搜索会议标题、总结和转写内容。
- 按时间或标题排序。
- 删除本地会议记录。

历史记录保存在本机。删除后不会自动从后端恢复。

## 9. 设置

设置页包含：

- 后端服务地址配置。
- 连接测试。
- 清除本机后端配置。
- 密钥策略说明。
- AI 处理链路说明。

注意：设置页不会保存 DashScope、OSS 或模型 API Key。

## 10. 使用建议

- 长会议建议连接稳定 Wi-Fi。
- 录音前先测试后端连接。
- 多人会议尽量让发言间隔清晰，便于说话人分离。
- 重要会议处理完成后，建议复制总结或导出保存。
- 公开部署后端时不要裸露接口，先加网络访问控制或后续实现 App 端后端 Token。
