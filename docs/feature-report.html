RecordWise 二开版功能报告

会议录音、转写、总结和待办的一体化 App

本报告基于当前仓库实现整理，覆盖 Flutter 移动端、FastAPI 后端、DashScope/Qwen/OSS AI 链路、开源分发配置方式和当前风险边界。

生成日期：2026-06-21 前端：Flutter Android 后端：FastAPI AI：DashScope + Qwen

一、产品定位

1

会议采集入口

支持现场录音和上传音频，适合会议、访谈、讲座、电话、聊天和头脑风暴场景。

3

结果页签

会议详情按原录音文本、总结文本、待办事项三类组织，减少用户查找成本。

5

翻译朗读语言

首版支持英语、日语、粤语、普通话、法语的指定发言翻译和 TTS 朗读。

二、核心工作流

1. 录音/上传手机端采集音频并提交到后端。
2. 音频处理后端标准化音频格式，准备上传。
3. OSS URL录音上传 OSS，生成 ASR 可访问的 URL。
4. AI 分析DashScope ASR 说话人分离，Qwen 生成总结和待办。
5. App 展示移动端保存结果并展示三类会议文档。

三、功能清单

模块 已实现能力 用户价值
首页 启动导航页、首页视觉图、最近会议入口、录音入口。 降低首次使用理解成本，快速进入会议处理。
录音页 现场录音、上传音频、会议类型选择、主要语种选择、最长录音提示。 覆盖现场会议和已有录音两种采集方式。
原录音文本 按说话人和时间展示转写段落，说话人改名，头像自动匹配，指定发言翻译朗读。 方便回溯会议原话，减少人工整理时间。
总结文本 总结编辑、复制、模板导入、模板分析确认、按模板重新生成、原文汇总子页签。 适配公司内部固定纪要格式，保留可编辑交付物。
待办事项 待办列表、重点标注、完成勾选、完成后删除线。 让会议动作项可以被追踪，而不只是生成文字。
历史记录 本机保存、搜索、排序、删除历史会议。 支持会后查阅和复盘。
设置 后端服务地址输入、保存、测试连接、清除本机配置、密钥策略说明。 适合开源分发，用户使用自己的后端和自己的 API 额度。

四、AI 能力链路

转写与说话人分离

  • 服务：阿里云 DashScope Fun-ASR / Paraformer。
  • 能力：音频转文字、说话人分离、时间段返回。
  • 依赖：后端上传 OSS 后提供可访问音频 URL。

总结和待办

  • 服务：Qwen 文本模型。
  • 能力：按会议纪要格式输出总结、提取 action items。
  • 规则：同一事项被 3 个或以上不同说话人提到才高亮为重点待办。

翻译朗读

  • 服务：Qwen-MT + Qwen-TTS。
  • 范围：英语、日语、粤语、普通话、法语。
  • 方式：用户在原录音文本中选择某段发言后生成翻译和音频。

模板化纪要

  • 用户粘贴模板文本。
  • App 分析模板结构并要求确认。
  • 后端按模板重新生成总结文本。

五、开源分发配置模型

当前改造后的推荐模式是：仓库公开代码，用户自行部署后端并配置自己的 DashScope / OSS 密钥，APK 只填写用户自己的后端服务地址。这样不会出现所有人共用开发者 API 额度的问题。

配置项 保存位置 原因
后端服务地址 手机本地 Hive 设置 每个用户的部署地址不同，需要 App 可编辑。
DashScope API Key 后端 `.env` 模型密钥不能进入 APK，避免泄漏和滥用。
阿里云 OSS AccessKey 后端 `.env` OSS 上传属于服务端能力，不能暴露到客户端。
模型名称 后端 `.env` 便于后端统一升级模型，不需要重新发 APK。
